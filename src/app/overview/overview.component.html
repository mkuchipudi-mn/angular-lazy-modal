Overview

This is demonstration of few possible ways we can route modals. We have an examples using wrapper components loaded into router-outlet and one without.

Handling of opening modals can also be moved to either router guards or resolvers, however both to those approaches will also require defining empty routes in loaded module.

Closing modal on navigation

One of the common patterns we have is closing modals on navigation. We may be able to handle this globally by simply listening for navigation changes and closing any qualied dialogs. See app.effects.ts. In this case we are just using NgRx effects and its router state, but any global servie might suffice.
